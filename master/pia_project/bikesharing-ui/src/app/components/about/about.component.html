This work is a semester project for KIV/PIA-E @ Západočeská Univerzita v Plzni

Use case

Bikesharing project based on Task assignment

Application offers several pages to achieve the task. The pages are described bellow.

Communication with the user is done through either page elements (HTML), or through pop out dialogs (toast).

Pages
Bikesharing (homepage)

Contains a map provided by OpenStreetMap.

On this map stands, rides and rides in progress are shown using markers.

Logged in users will also see a row with 2 select boxes and a button to start a new ride above the map.

Stands - anyone can see them. Uses square markers with numbers. Numbers are a count of how many bikes are currently stored at the stand. Markers with bikes are cyan and markers with no bikes are orange.

Green bike marker - ride of the currently logged in user. This marker tries to follow a path, that is highlighted as blue path along the roads.

Gray bike marker - ride of different user that is being tracked to show the bike location.

Ride row - user cannot start a ride from location with no bikes. User can't select the same location in both select boxes.

Finished ride row - while ride is in progress it shows target location and approximate distance. Finishing the ride is done automatically upon reaching the destination.

Past rides

Simple table showing past rides of logged in user.

Due for service

Simple table showing bikes that require service. Service interval is configurable and is set on the backend.
